Add metadata-driven signal path diagram visualization to SpinGPT using Cytoscape.js with Dagre layout. Users can see auto-laid-out block diagrams showing signal flow when valid ;@fx metadata exists in their .spn code. Tool remains fully functional without metadata (graceful degradation).

Purpose: Provide visual documentation of signal flow to help users understand complex effects and feedback paths without interpreting code.

Output: Collapsible diagram section in simulation panel that renders analog-styled block diagrams from metadata, with feedback cycles visually distinguished via dashed edges.
</objective>

       - Export `FxMetadata` interface matching schema (effectName, io, pots, memory, graph)
       - Export `extractMetadata(source: string): FxMetadata | null` function
       - Parse ;@fx v1/v2 headers from SpinASM source using regex to find lines starting with `;@fx`
       - Extract JSON content between opening `{` and closing `}` after `;@fx` markers
       - Handle multi-line JSON (accumulate lines until valid JSON parsed)
       - Return null if no metadata found or JSON parse fails (graceful degradation)
       - Do NOT throw errors - return null for any parsing issues
    
    3. Create `src/utils/graphBuilder.ts`:
       - Export `buildCytoscapeElements(metadata: FxMetadata): cytoscape.ElementDefinition[]`
       - Convert `metadata.graph.nodes` array (strings) to Cytoscape node elements with `id` and `label`
       - Infer node `type` from label keywords: "delay" → "delay", "filter"/"lpf"/"hpf" → "filter", "reverb" → "reverb", "mix" → "mixer", "input" → "input", "output" → "output", default → "mixer"
       - Convert `metadata.graph.edges` to Cytoscape edge elements with `source` and `target`
       - Detect feedback cycles: Use simple heuristic (edge from higher topological rank to lower rank)
       - Mark feedback edges with `classes: 'feedback'` for styling
       - Return empty array if metadata.graph is missing or invalid
  </action>
