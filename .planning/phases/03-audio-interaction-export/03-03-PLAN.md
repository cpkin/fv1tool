  <action>
Create AnalogKnob.tsx component with analog aesthetic and flexible interaction:

1. Props:
   - value: number (0-11 range)
   - onChange: (value: number) => void
   - label: string (e.g., "POT0")
   - disabled: boolean (during re-render)

2. State:
   - isDragging: boolean
   - isEditing: boolean
   - editValue: string
   - dragStartY: number (for vertical drag)
   - dragStartAngle: number (for circular drag)

3. Visual rendering:
   - Circular knob div with CSS transform rotate based on value
   - Rotation range: -135° (value=0) to +135° (value=11)
   - Indicator line from center to edge showing current position
   - Numeric value display below knob: value.toFixed(1)
   - Label below value

4. Vertical drag interaction:
   - onMouseDown: capture starting Y position, set isDragging
   - onMouseMove: calculate delta = startY - currentY (up = increase)
   - Update value: newValue = oldValue + (delta * 0.02)
   - Clamp to [0, 11], call onChange

5. Circular drag interaction:
   - onMouseDown: capture starting angle from center (Math.atan2)
   - onMouseMove: calculate current angle, derive rotation delta
   - Convert angle to value in 0-11 range (270° rotation span)
   - Use whichever has larger movement (vertical dy vs horizontal dx) to pick mode

6. Inline editing:
   - onClick (on value display): enter edit mode, show input field
   - Input accepts numeric entry
   - onBlur or Enter: parse value, clamp to [0, 11], call onChange
   - Invalid input: revert to previous value

7. Disabled state:
   - Gray out knob, show cursor not-allowed
   - Ignore all mouse events

Style in app.css:
- Knob: 80px diameter circle, dark background (#2a2a2a), border (#444)
- Indicator: 3px wide line from center to edge, bright color (#3b82f6)
- Rotate transform on knob element (not indicator)
- Value display: monospace font, 18px
- Label: 12px, gray text
- Hover: subtle glow effect

  <action>
Enable fast re-render by caching compiled instructions and reusing them on knob change:

1. Update audioStore.ts state:
   - Add cachedInstructions: CompiledInstruction[] | null
   - Add cachedIOMode: IOMode | null
   - Add cachedInputBuffer: AudioBuffer | null (store decoded/resampled input)

2. Add actions:
   - setCachedRender(instructions, ioMode, inputBuffer)
   - clearCachedRender() (reset all cached fields to null)

3. Update renderSimulation.ts flow (or create wrapper in SimulationPanel):
   - After successful parse/compile on first render: store instructions, ioMode, inputBuffer in cache
   - On subsequent renders (knob changes): check if cache exists
   - If cached: skip parse/compile, directly call interpreter with cached instructions + new pot values
   - If not cached (new code uploaded): full parse/compile/render flow

4. Performance target:
   - First render: full pipeline (parse → compile → render) ~2s for 30s audio
   - Re-render with cache: interpreter only (skip parse/compile) should be 30-50% faster

5. Cache invalidation:
   - Clear cache when: editor code changes, new audio uploaded, IO mode changes
   - Keep cache when: only POT values change

6. Add timing instrumentation:
   - Log "Re-render using cached instructions: Xms" to console
   - Compare to full render timing from Phase 2

     ```typescript
     if (audioStore.cachedInstructions) {
       await renderSimulation({
         instructions: audioStore.cachedInstructions,

         inputBuffer: audioStore.cachedInputBuffer,
         pots: audioStore.pots // updated knob values
       })
     }
     ```

   - Disable knobs during render (prevents queue buildup)
